Attach IAM Policy to IAM User
--
Why to attach IAM Policy
Attaching an IAM policy to an IAM user is essential 
to grant specific, authorized permissions for accessing and managing AWS services and resources.

Checklist for implementation
1) Sign in to AWS: Access the AWS Management Console and navigate to the IAM dashboard.
2) Locate User: Click Users in the left navigation pane and select the specific user name.
3) Add Permissions: Click Add permissions and select Add permissions from the dropdown menu.
4) Attach Policy: Choose Attach existing policies directly.
5) Search and Select: Use the search box to find your desired policy, then check the box next to the policy name.
6) Review and Save: Click Next: Review, and finally, select Add permissions to apply the policy.

Note:Policies define what actions (e.g., read, write) a user can perform on specific resources.
