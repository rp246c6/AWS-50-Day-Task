Snapshot and Restoration of an RDS Instance
--
why to take snapshot and restore RDS instance
IT ensures data durability, enables disaster recovery,
allows testing on production data, and protects against accidental deletion or corruption.

checklist:
1)Set Variables
2)Ensure the source RDS instance is in available state.
3)Take Snapshot
4)Restore Snapshot to New RDS Instance.
5)Validate RDS restoration.
